Search Filters component

This component is used to trigger searches of cases. The elements of the contract are:

  • Input fields:
    • autoApply - a flag indicating a search should happen as part of the loading of the component
  • Output fields:
    • onApply - a notification event about an apply button being clicked containing 
                                              {{selected}}
      where:
      • caseType - the selected case type
      • formGroup - angular form group containing all configured search input fields
      • jurisdiction - the selected jurisdiction
      • metadataFields - an array containing all configured the metadata fields
      • page - the number of the page in search results
    • onReset - a notification event about a reset button being clicked
    • onJuridiction - a notification event with selected jurisdiction object

Additionally the component saves user selections to local browser storage any time a user click Apply button after selecting dropdown values. This way any time a user is back on the Create Case Filters page the dropdown will be restored from the last applied selection. This can be cleared by clicking Reset button.
